Add a node to the dependency graph. If the node already exists, update the properties of the node. If the node doesn’t exist, create a new node. If the node belongs to a package group, add an edge between the package parent and the new node.

Parameters:

  • g – The dependency graph

  • n – The new node to add to the graph

  • groups – The groups of packages to modules in the package

  • edge_settings – If an edge is to be added, the settings for that edge

  • node_settings – If a node is to be added, the settings for that node

Returns:

None

evase.structures.analysisperformer.extend_depgraph_attackvectors(graph: DiGraph, groups: Dict[str, Set[str]], analysis: Dict[str, DiGraph]) → Tuple[DiGraph, Dict[str, Set[str]]][source]¶

Extend the main dependency NetworkX graph with the subgraphs for each individual vulnerability.

Basic Idea: - Go through each vulnerability graph found during the analysis and add all nodes and edges from these subgraphs to the main graph - Copy the vulnerability graphs to the main graph

Parameters:

  • graph – The main dependency graph

  • groups – Groups in the graph (similar modules together)

  • analysis – The analysis results of the program, keys are modules and values are the NetworkX graphs

Returns:

The extended graph and new groupings
